## Blocking Issues (FAIL)

### [CODE-5 / SUPPLY-2] `xlsx` dependency: 2 unpatched high-severity runtime vulnerabilities
**Evidence:** `npm audit` output:
```
xlsx  *
Severity: high
Prototype Pollution — GHSA-4r6h-8v6p-xvw6
ReDoS              — GHSA-5pgg-2g8v-p4x9
No fix available
```
`xlsx` is a **production runtime dependency** (`dependencies`, not `devDependencies`), imported in `src/lib/fileExtract.ts:64` for `.xlsx`/`.xls` file parsing. Prototype Pollution allows user-supplied spreadsheets to corrupt the JavaScript object prototype; ReDoS allows a crafted spreadsheet to hang the JS thread. Both are triggered by parsing user-uploaded files — a direct attack surface. `xlsx` is effectively abandonware with no maintained fix.

**Fix:** Replace `xlsx` with `exceljs` (active maintenance, no known high CVEs) or route through the existing Gemini `/api/extract-file` flow (already handles `.xlsx` server-side). Drop the `xlsx` package entirely if the server route is sufficient.

---

### [CODE-5b] `vite-plugin-pwa` / `workbox-build`: 4 additional high-severity vulnerabilities
**Evidence:** `npm audit` shows `workbox-build` → `@rollup/plugin-terser` chain. These are **build-time only** (devDependencies), so they do not ship to production browsers. FAIL because the CI pipeline fails `npm audit --audit-level=high`.

**Fix:** Upgrade `vite-plugin-pwa` when a patched version is available, or document build-only risk acceptance.

---

### [DATA-5] `api/transfer.ts` is missing — `/api/export` and `/api/import` return 404 in production
**Evidence:**
- `vercel.json` rewrites: `"/api/export"` → `"/api/transfer"` and `"/api/import"` → `"/api/transfer"`
- `api/transfer.ts` does **not exist** anywhere in the repo
- `src/components/settings/DangerTab.tsx:50` calls `/api/export` for the "Export All Data" delete-account flow
- `src/components/settings/BrainTab.tsx:13` uses `/api/export` for brain backup

Both export buttons silently fail in production. The **right-to-erasure export** (required under GDPR/POPIA before account deletion) is broken.

**Fix:** Implement `api/transfer.ts` with `GET /api/export?brain_id=` (returns entries as JSON) and `POST /api/import` (bulk capture), following the existing auth/rateLimit/checkBrainAccess pattern.

---

### [DATA-1] No privacy policy
**Evidence:** No privacy policy route, page, or link found anywhere in the codebase or `public/` directory. Sentry is configured with `sendDefaultPii: true` (`src/main.tsx:12`) which sends user IP addresses and email addresses to Sentry without documented user consent. User data also flows to Google Gemini (entry content), Groq (audio), and Vercel.

**Fix:** Add a privacy policy page linked from the login screen. Set `sendDefaultPii: false` in Sentry unless explicit consent is obtained. Required for GDPR/POPIA compliance.

## Warnings (WARN)

### SECURITY
- **[S1-1] Sentry DSN hardcoded in `src/main.tsx:11`** — Move to `VITE_SENTRY_DSN` env var to allow rotation without a code deploy.
- **[S1-3] HSTS header missing** — `vercel.json` sets CSP, X-Frame-Options, and other headers but no `Strict-Transport-Security`. Add: `{ "key": "Strict-Transport-Security", "value": "max-age=31536000; includeSubDomains; preload" }`.
- **[S1-6] Rate limiting is per-instance (in-memory) if Upstash not configured** — `api/_lib/rateLimit.ts` explicitly notes: "in-memory (per-instance, zero real protection in serverless)". Verify `UPSTASH_REDIS_REST_URL` and `UPSTASH_REDIS_REST_TOKEN` are set in production.
- **[S1-12] Auth tokens pass through URL hash post-magic-link** — `src/App.tsx:16-24` extracts tokens from `window.location.hash`. Supabase standard pattern; verify token TTL is ≤5 minutes in Supabase dashboard.

### DATABASE
- **[DB-1] Backup restoration untested** — Supabase paid plans include daily backups; verify restore has been tested at least once.
- **[DB-3] Cannot confirm dev/prod DB separation** — Verify dev and production point to different Supabase projects.
- **[DB-8] User API keys stored as plaintext** — `groq_key` and `gemini_key` in `user_ai_settings` table. Consider Supabase Vault or column-level encryption.
- **[DB-9] `sendDefaultPii: true`** sends user email/IP to Sentry — see DATA-1 FAIL.

### DEPLOYMENT
- **[DEP-1] Cannot verify all env vars are set in production** — Required: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `GEMINI_API_KEY`, `GROQ_API_KEY`, `GEMINI_MODEL`, `UPSTASH_REDIS_REST_URL`, `UPSTASH_REDIS_REST_TOKEN`, `CRON_SECRET`, `VITE_APP_URL`. Run `vercel env ls` to audit.
- **[DEP-5] No documented rollback procedure** — Vercel has one-click rollback; document it in a runbook.
- **[DEP-6] No staging environment visible** — CI only targets `main`. Add a `staging` branch for pre-production validation.

### CODE QUALITY
- **[CODE-1] Extensive `console.log`/`console.error` throughout API and frontend** — Includes audit-trail entries like `[audit] HARD_DELETE entry id=... user=...` (e.g. `api/entries.ts:242`). Structured JSON logging would be searchable by log aggregators.
- **[CODE-3] Loading/error states in UI** — Cannot fully verify without browser testing.
- **[CODE-10] Feature flag `VITE_ENABLE_MULTI_BRAIN`** in `src/lib/featureFlags.ts` — Document owner and intended removal date.

### PERFORMANCE
- **[PERF-2] Large bundles: `pdfjs-dist`** (~4 MB) and **`xlsx`** (~3 MB) — `xlsx` is lazy-imported which is good. Verify `pdfjs-dist` is also lazy-loaded. Consider routing all PDF/Excel extraction through the server-side Gemini endpoint and removing both from the client bundle.
- **[PERF-9] N+1 query in `api/cron/gap-analyst.ts`** — Fetches all brains then loops to fetch entries per brain. O(n) DB calls; acceptable for a weekly cron with few users but will degrade at scale.

### SEO & META
- **[SEO-1] No `<meta name="description">` in `index.html`.**
- **[SEO-2/3] No `robots.txt` or `sitemap.xml`** in `public/` — Crawlers will attempt to index the auth redirect wall.
- **[SEO-5] No Open Graph tags** — Social sharing shows no preview card.

### OBSERVABILITY
- **[OBS-2] No external uptime monitoring** — `/api/health` endpoint is comprehensive (checks DB + Gemini + Groq) but nothing external pings it. Add UptimeRobot or Checkly.
- **[OBS-3] Unstructured logging** — See CODE-1 above.

### API HARDENING
- **[API-3] No explicit Vercel function body size config** — Add `export const config = { api: { bodyParser: { sizeLimit: "5mb" } } }` to LLM and extract-file endpoints to enforce limits before parsing.

### DATA & PRIVACY
- **[DATA-2] No cookie consent banner** — Verify whether Sentry and Vercel Speed Insights set cookies; add POPIA-compliant consent if so.
- **[DATA-3] No automated data purge** — No cron job purges entries where `deleted_at < NOW() - 30 days`.
- **[DATA-7] No Terms of Service** — Not found in codebase or public directory.

### FRONTEND HARDENING
- **[FH-3] Supabase auth token in `localStorage`** — Supabase JS v2 default; acceptable given tight CSP, but document as accepted XSS risk.
- **[FH-4] No SRI on Google Fonts / Material Symbols** — `index.html:13-15` loads from `fonts.googleapis.com` without `integrity` attributes.

### BUSINESS CONTINUITY
- **[BC-1–6]** Incident response plan, on-call rotation, status page, disaster recovery runbook, and contact documentation are absent. Recommend a `docs/runbook.md` at minimum.
