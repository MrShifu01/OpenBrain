# Sprint 3 — Making OpenBrain Actually Useful Daily

The focus of this sprint is shifting from "captures well" to "surfaces the right info at the right time." Everything here is about real daily value, not technical improvements.

---

## 1. Proactive Intelligence Engine

**Goal:** The app comes to you — not the other way around.

### What to build:
- Context-aware nudges generated by Haiku on each app open (not just Digest view)
- Stale entry detection: ideas stuck on "concept" for 2+ weeks, reminders with no action
- Gap detection: "You have a restaurant but no lease/insurance/fire certificate captured"
- Smart merge suggestions: "Delta Distribution and Delta Gas — same company?"
- Action prompts: "Your driving licence renewal is in 4 months — here's what you need to do"
- Nudges appear as a dismissable banner at the top of the Grid view

### Implementation notes:
- On app load, send 30 most recent entries + links to Haiku with a "what should Chris know right now?" prompt
- Cache the nudge in sessionStorage so it doesn't re-fetch on every tab switch
- Limit to 1-2 nudges at a time — more than that gets ignored

## 2. Supplier Quick-Access Panel

**Goal:** One tap to see supplier name, what they supply, and their phone number.

### What to build:
- New "Suppliers" section on Grid view (or dedicated sub-tab) that filters entries tagged as suppliers
- Each supplier card shows: name, what they supply, phone number (pulled from metadata/content), last order date if tracked
- Tap-to-call / tap-to-WhatsApp buttons on each card
- "Reorder" quick action that creates a reminder entry: "Reorder from [supplier] — [date]"

### Implementation notes:
- Filter by `metadata.category === "supplier"` or tags containing "supplier"
- WhatsApp link: `https://wa.me/27XXXXXXXXX` (strip leading 0, add country code)
- Phone link: `tel:XXXXXXXXXX`

## 3. Cost & Price Tracking on Entries

**Goal:** Build a cost database for the business without extra effort.

### What to build:
- Add optional `price` and `unit` fields to metadata (e.g. "R85/kg", "R120/case")
- AI prompt updated to extract prices when mentioned: "mince balls R85 per kg" → `metadata.price: "R85", metadata.unit: "per kg"`
- Supplier entries show price inline on cards
- Simple cost summary view: list all entries with prices, grouped by supplier

### Implementation notes:
- Update the classification system prompt to include price extraction rules
- No new DB columns needed — prices go in the existing metadata JSONB field

---

## 4. Quick Actions on Entries

**Goal:** Entries lead to actions, not just storage.

### What to build:
- Context-sensitive action buttons in the Detail Modal based on entry type:
  - **Supplier**: "Call", "WhatsApp", "Reorder"
  - **Reminder**: "Mark Done", "Snooze 1 week", "Snooze 1 month"
  - **Idea**: "Start this" (moves status from concept → in progress), "Archive"
  - **Person**: "Call", "WhatsApp", "Send info to someone"
  - **Document**: "Set renewal reminder"
- "Mark Done" on reminders updates `metadata.status` to "done" and drops importance to 0
- "Snooze" pushes the deadline/due_date forward

### Implementation notes:
- Actions are just targeted `onUpdate` calls with specific metadata changes
- Phone/WhatsApp actions extract numbers from metadata or content using regex

## 5. "Who Do I Call?" Shortcut

**Goal:** Instant answer to "who can fix X?" or "who supplies Y?"

### What to build:
- Shortcut button in the Chat view: "Who do I call for..."
- Pre-fills the chat with the question pattern and sends immediately
- AI responds with name + phone number + context from the entries
- Results show tap-to-call buttons inline in the chat response

### Implementation notes:
- Add 3-4 quick-ask chips above the chat input: "Who supplies...", "When does... expire?", "What's the number for..."
- Parse AI response for phone numbers and render them as clickable links

## 6. Business vs Personal Workspace Toggle

**Goal:** Reduce noise when in work mode.

### What to build:
- Toggle at the top of Grid view: "All" | "Business" | "Personal"
- Auto-classify entries as business/personal based on tags and content:
  - Business: anything tagged with "smash burger bar", "supplier", "contractor", or type is supplier/business contact
  - Personal: identity documents, health, family, personal contacts
  - Both: reminders, ideas (can go either way)
- AI classification prompt updated to include `workspace: "business" | "personal"` in output
- Persist last-used workspace in localStorage

### Implementation notes:
- Add `workspace` to the AI classification JSON format
- For existing entries, infer workspace from tags (anything with "smash burger bar" = business)

## 7. Pre-Save Preview & Duplicate Check

**Goal:** Build trust that the AI is getting it right, and prevent duplicate mess.

### What to build:
- After AI classifies a quick capture, show a preview card before saving:
  - Title, type, importance, tags — all editable inline
  - "Save" or "Edit before saving" buttons
  - If a similar entry exists, show: "You already have '[title]' — update it instead?"
- Duplicate detection: compare new entry title against existing entries using fuzzy matching (reuse `scoreEntry`)
- If score > threshold, show the existing entry and offer: "Save as new" | "Update existing" | "Cancel"

### Implementation notes:
- This changes the QuickCapture flow from instant-save to preview-then-save
- Keep a "quick mode" toggle for users who want instant capture without preview
- Duplicate threshold: scoreEntry > 50 on title alone

## 8. Undo System

**Goal:** Confidence to act quickly without fear of losing data.

### What to build:
- Toast-based undo for destructive actions:
  - "Entry deleted — Undo" (5 second window)
  - "Entry updated — Undo" (5 second window)
  - "Entry created — Undo" (3 second window)
- Undo restores the previous state from the action
- Only tracks the last action (not a full undo stack)

### Implementation notes:
- Store `lastAction` in state: `{ type: "delete"|"update"|"create", entry: {...}, previous: {...} }`
- On undo: reverse the action (re-add deleted entry, restore previous version, remove created entry)
- Toast auto-dismisses after timeout
- Delete undo: don't actually call the API delete until the toast expires — optimistic deletion with deferred commit

## 9. Morning Briefing Notification

**Goal:** Make OpenBrain part of the daily routine without opening the app.

### What to build:
- Service worker push notification at a configurable time (default 7:00 AM)
- Content: "2 reminders this week | 1 expiring document | 3 new connections found"
- Tapping the notification opens the Digest view
- Settings toggle: enable/disable, set time

### Implementation notes:
- Requires Push API + Notification API permissions
- Service worker checks reminders from cached localStorage data
- Use `self.registration.showNotification()` in the service worker
- Schedule via `setTimeout` calculated from current time to target time
- PWA manifest already exists — just needs push handler in the service worker

## 10. WhatsApp / Voice Capture

**Goal:** Capture info without typing on a phone.

### What to build:
- **Voice capture**: microphone button next to the camera button in QuickCapture
  - Uses Web Speech API (`SpeechRecognition`) to transcribe
  - Transcribed text goes into the input field, then processed normally by AI
  - Fallback: record audio → send to Haiku for transcription (via audio content type)
- **WhatsApp forwarding** (future/manual):
  - Document a flow: forward WhatsApp message to a dedicated number/email → webhook → capture API
  - For now, add a "Paste from WhatsApp" hint in the capture bar

### Implementation notes:
- `webkitSpeechRecognition` works in Chrome/Edge/Safari on mobile
- Set language to `en-ZA` for South African English
- Show live transcription as the user speaks
- Auto-submit after 2 seconds of silence

---

## 11. Shareable Entries

**Goal:** Send a supplier's number to your manager, or share the supplier list with a partner.

### What to build:
- "Share" button in the Detail Modal
- Uses Web Share API (`navigator.share()`) on mobile — falls back to clipboard copy on desktop
- Share formats:
  - Single entry: formatted text with title, content, metadata, phone numbers
  - Filtered set: "Share all suppliers" exports a clean list
- Share as text (WhatsApp-friendly format):
  ```
  Ehrlichpark Butchery - Mince Balls
  Supplies mince balls for smash burgers. Key ingredient.
  Location: Bloemfontein
  — from OpenBrain
  ```

### Implementation notes:
- Web Share API is supported on all modern mobile browsers
- Desktop fallback: copy to clipboard + toast "Copied to clipboard"
- For filtered sharing, add a "Share list" button to the Grid view when a type filter is active
