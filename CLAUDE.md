## Development Practices

### Test-Driven Development (TDD)

All code work on this project must follow red-green TDD:

1. **Red** — Write a failing test first that defines the desired behavior.
2. **Green** — Write the minimum code necessary to make the test pass.
3. **Refactor** — Clean up the code while keeping all tests green.

No production code should be written without a corresponding failing test written first.
